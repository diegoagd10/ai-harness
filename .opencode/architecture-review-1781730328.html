Collapse the three-loop install/uninstall dispatch

Strong in-process

After

flowchart TB
  Inst["install() / uninstall()"]
  Inst --> Disp["registry dispatch"]

Problem: install() and uninstall() carry three near-identical try/except loops — one per artifact kind. The dispatch on artifact type is shallow: same try/except, same place/remove pattern, only the prepare step varies.

Solution: Single dispatch through a per-kind registry. Each artifact kind contributes its own (prepare_fn, place_fn, remove_fn). The two public functions iterate one list; the loop body is shared.

  • locality: changes to how ComposedFileArtifact is installed touch one place.
  • leverage: one dispatch reached by all three CLIs (copilot, opencode, claude).
  • interface unchanged: install(manifest, home, console) stays.
  • tests: each kind has its own contract; new kinds register and inherit the shared error policy.

Problem: copilot.py:_build_manifest() has two near-identical loops for SDD-phase and inline JD/reviewer agents. Only the body_source path varies. Adding a third category forces another near-identical loop.

Solution: Single loop over an iterable of (name, body_source) pairs. _PHASE_NAMES and _INLINE_AGENTS each contribute their resolver; the manifest builder iterates one list.

  • locality: composed-agent construction in one place.
  • leverage: a new agent category touches the resolver, not the manifest builder.
  • interface unchanged: _build_manifest(home) stays.

Problem: _place_file and _remove_file thread backup_suffix and conflict_suffix separately. The rotation policy is implicit across two functions.

Solution: Extract a BackupPolicy dataclass that owns both suffixes and the rotation rule. Both functions take a policy; per-artifact override still possible via dataclass default.

  • locality: rotation rule in one place.
  • leverage: future artifact kinds reuse the same policy object.

Normalize the template field default

Speculative in-process
src/ai_harness/artifacts/manifest.py

Problem: FileArtifact.template defaults to None; ComposedFileArtifact.template defaults to {}. Same concept, two defaults.

Solution: Pick one default — field(default_factory=dict) on both. None complicates the substitution loop with a None-check.

  • locality: template substitution is one concept, one default.

Top recommendation

Collapse the three-loop install/uninstall dispatch. Highest leverage: the duplicated loops live in installer.py and run on every install for every CLI (copilot, opencode, claude). Replacing them with a per-kind registry keeps the public interface (install(manifest, home, console)) unchanged while concentrating the I/O policy in one dispatch.
